# README -- Carpeta /otros

## 1. Propósito de esta Carpeta

La carpeta **/otros** contiene documentación y archivos que **no
pertenecen** a las siguientes categorías del proyecto:

-   Documentación formal del proyecto
-   Material académico o universitario
-   Implementaciones o recursos relacionados con RAG
-   Investigación estructurada o fuentes bibliográficas

Su función es actuar como un espacio controlado para contenido
complementario que no encaje en las áreas anteriores.

------------------------------------------------------------------------

## 2. Qué DEBE ir en /otros

-   Notas personales o exploratorias.
-   Pruebas conceptuales aisladas.
-   Recursos auxiliares temporales.
-   Material experimental que aún no forma parte del marco formal del
    proyecto.
-   Archivos de contexto general no vinculados directamente a
    universidad, investigación o RAG.

------------------------------------------------------------------------

## 3. Qué NO debe ir en /otros

No deben colocarse aquí:

-   Documentos oficiales del proyecto.
-   Referencias bibliográficas o marco teórico.
-   Material académico evaluable.
-   Documentación técnica de arquitectura.
-   Implementaciones o configuraciones relacionadas con RAG.

Si un archivo adquiere relevancia estructural o académica, debe moverse
a la carpeta correspondiente.

------------------------------------------------------------------------

## 4. Criterio de Organización

Esta separación evita ambigüedad, mantiene claridad metodológica y
facilita la auditoría del repositorio. La carpeta /otros no es un
repositorio desordenado, sino un espacio delimitado para contenido fuera
del alcance formal del proyecto.
