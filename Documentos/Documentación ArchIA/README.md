# README -- Documentación Técnica del Proyecto

## 1. Propósito

Esta carpeta contiene la documentación técnica formal del sistema
desarrollado en el proyecto. Su objetivo es describir de manera
estructurada, reproducible y auditable la arquitectura, decisiones de
diseño, componentes y funcionamiento interno del sistema.

------------------------------------------------------------------------

## 2. Alcance

Aquí deben incluirse exclusivamente documentos relacionados con:

-   Arquitectura del sistema.
-   Diagramas (C4, componentes, secuencia, despliegue).
-   Decisiones de diseño (ADR -- Architecture Decision Records).
-   Especificaciones técnicas.
-   Modelado de datos.
-   Integraciones entre módulos.
-   Consideraciones de seguridad y escalabilidad.
-   Requisitos funcionales y no funcionales.
-   Documentación de APIs internas.

------------------------------------------------------------------------

## 3. Qué NO debe ir aquí

No deben colocarse en esta carpeta:

-   Referencias bibliográficas o marco teórico (van en investigación).
-   Documentación administrativa universitaria.
-   Notas exploratorias no formalizadas.
-   Implementaciones de código sin documentación técnica asociada.

------------------------------------------------------------------------

## 4. Principios de Documentación

Todo documento técnico debe:

1.  Explicar el problema que resuelve.
2.  Justificar la decisión adoptada.
3.  Describir alternativas consideradas.
4.  Incluir diagramas cuando sea pertinente.
5.  Permitir reproducibilidad técnica.

La documentación técnica no es descriptiva superficial; debe ser
estructural y justificativa.
